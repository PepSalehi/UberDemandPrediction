Overview

This site presents a Methodology, the Demand Prediction Results and an API to predict demand for Uber services in Washington D.C. from May 1, 2012 through May 15, 2012 as part of the Demand Prediction Challenge.


Methodology

I present a simple methodology that takes the hits from mobile apps and crunches the data to find patterns in the raw stream of data from one market. Using Uber App useage to reveal insights into human behavioral patterns, habits and movement in Washington DC.



Results

How well does the model predict the demand in the future? Answers to When? and How Much? all from a simple timestamp in UTC from a mobile phone.



Application Programming Interface (API)

Supercharge a demand prediction algorithm then share it with the world. A streaming API that takes queries in YYYY-DD-MMTHH:MM:SS format and returns a JSON object with prediction data and other ancillary information.
